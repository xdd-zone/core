# Console 设计上下文

这份文档只给 `packages/console` 用。页面、布局、导航、列表、表单、空状态都按这里的方向处理。

## 先记住当前产品气质

- 这是后台，不是展示站
- 重点是操作效率，不是视觉表演
- 页面要清楚、稳定、耐看
- 有内容就直接展示，没内容就别硬补装饰块

## 主要使用者

当前主要使用者是后台管理员。
他们进入页面后，最关心的是：

1. 这页能做什么
2. 现在结果是什么
3. 下一步点哪里

## 当前默认风格

### 页面气质

- 克制
- 清楚
- 安静
- 有秩序

### 不要做成什么样

- 不做欢迎页风格的大首页
- 不做一堆没有真实数据的统计卡
- 不做很重的渐变、发光、玻璃效果
- 不靠大图标和大标题撑页面

## 页面结构规则

### 顶部区域

默认只放这些内容：

1. 页面标题
2. 一句很短的说明
3. 少量摘要标签或关键数字

默认不要：

- 眉题
- 标题左侧大图标
- 很长的介绍文字
- “欢迎回来”“一目了然”这类气氛文案

### 主内容区

页面主体要直接服务当前操作。

常见结构：

- 列表页：标题 + 筛选区 + 表格
- 编辑页：标题 + 当前对象信息 + 表单
- 自助页：标题 + 摘要标签 + 结果区
- 首页：标题 + 常用入口 + 当前账号信息

如果主内容已经很明确，就不要再塞很多说明块。

## 信息密度

- 保持紧凑，但不要挤
- 页面模块之间留出清楚的层级
- 解释性文字尽量少于展示性内容
- 一句话能说清，就不要写成一段

## 列表页怎么做

列表页的重点永远是表格和筛选，不是顶部装饰。

默认结构：

1. 标题
2. 一句短说明
3. 右侧摘要标签
4. 筛选区
5. 表格

要求：

- 顶部面积不要抢主内容
- 不在表格上方堆很多解释
- 筛选项先回答“怎么缩小范围”
- 表格再回答“当前有哪些结果”

## 首页怎么做

当前默认首页是入口型首页，不是展示型首页。

先回答这三件事：

1. 当前账号是谁
2. 当前账号能看到什么
3. 现在最值得点哪个入口

没有真实业务数据时，不做：

- 趋势图
- 假统计卡
- 很大的欢迎区
- 纯展示用模块

## 详情页和编辑页怎么做

默认结构：

1. 标题
2. 当前对象摘要
3. 详情区
4. 编辑区或操作区

要求：

- 先给关键信息，再给详细信息
- 表单上方不要堆很多说明
- 表单标签已经说清的内容，不再重复解释

## 空状态和错误页怎么做

规则很简单：

- 先给结论
- 再给动作
- 文字尽量短
- 不做成海报

## 文案规则

页面文案统一按下面写：

- 直接
- 短句
- 不营销
- 不教学腔

优先用这些说法：

- 当前账号
- 当前范围
- 查看权限
- 编辑资料
- 角色列表
- 用户列表

少用这些说法：

- 帮助你快速了解
- 从这里开始
- 一目了然
- 全面掌握
- 为你整理

## 颜色和表面

统一沿用 Catppuccin，不开第二套主题系统。

优先用现有语义类：

- `bg-surface`
- `bg-surface-muted`
- `bg-surface-subtle`
- `text-fg`
- `text-fg-muted`
- `border-border`
- `border-border-subtle`
- `text-primary`
- `text-success`
- `text-warning`
- `text-info`
- `text-danger`

要求：

- 颜色主要用来表达层级和语义
- 同一种状态在不同页面保持一样
- 不直接临时写死一套新颜色

## 组件规则

### Ant Design

优先复用 Ant Design 组件，再用 Tailwind 语义类收样式。

### 表格

- 表格是列表页主角
- 表头、表体、分页、筛选区保持同一套层级
- hover 和选中反馈要轻

### 图标

- 统一用 Lucide
- 只在操作或局部语义里用
- 默认不要放在页面标题左边撑气氛

## 动效

只做轻量反馈：

- 页面进入
- hover / focus
- 展开收起
- 状态切换

不要做明显位移、缩放和跳动。

## 收尾检查

页面完成前，至少过一遍这几项：

- 标题是不是已经足够清楚
- 顶部是不是太重了
- 页面重点是不是落在真正能操作的内容上
- 有没有假的统计块或没用的说明块
- 文案是不是已经足够短
- 是否统一使用现有主题语义类
